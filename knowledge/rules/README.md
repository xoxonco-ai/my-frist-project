# 規則層：規則不是一句 prompt

同一批資料，套用不同規則，會產出完全不同的內容（客服回覆／銷售話術／公眾號文章）。規則要拆成以下維度分別定義，而不是塞進一句 prompt：

- **身份背景**：AI 扮演誰、代表什麼立場
- **目標讀者**：這份輸出是寫給誰看的
- **輸出格式**：長度、結構、標題規則
- **語氣風格**：正式/口語、繁體/簡體、品牌語氣
- **引用來源**：是否需要標注資料來源、怎麼標
- **禁用表達**：不能用的詞彙、句型、承諾
- **不知道就說不知道**：沒有依據時，AI 必須明確說不知道，不能編造

## 建議做法

每個任務類型（客服回覆／銷售話術／公眾號文章…）各自維護一份規則檔，放在這個資料夾下，例如：

- `customer-service.md`
- `sales-script.md`
- `article.md`

不要共用一份「萬用 prompt」，那樣沒辦法個別調整語氣和格式。
