# 索引層：AI 怎麼找到資料

原則：**先給路徑，再做檢索**。不要讓 AI 每次都全量翻資料。

隨著資料變多，依序升級以下四種做法：

## 1. 目錄說明（資料量小時）

直接告訴 AI：客戶問題、案例、產品資料分別放在哪個資料夾。最簡單、最穩定。

## 2. 主題地圖（資料開始分類）

用一張心智圖式的主題地圖，把 銷售話術／客戶問題／產品賣點／內容行銷 等主題彼此關聯起來，AI 靠主題找對應資料夾。

## 3. LLM Wiki（資料量變大）

每個主題建一份「知識指南」，包含：

- 總結
- 相關主題（連結）
- 來源位置
- 查詢範例（或常用關鍵字）

## 4. RAG（僅在資料量大、語意檢索困難時使用）

不是預設方案，是最後一步。資料量小或路徑穩定時，優先用 1–3。

## 警告

不要全量翻資料。路徑要穩定，AI 才能重複、可靠地找到同一批資料。
