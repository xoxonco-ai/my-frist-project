# 工作流層：進入真實任務

前三層（資料層→索引層→規則層）疊起來後，透過工作流層的橋樑，才真正「進入真實任務」——不是只做問答。

## 真實任務範例

- 寫文章
- 客戶問題
- 銷售話術
- 培訓新人
- 個人復盤

## 閉環：反饋回寫 → 人工驗收 → 跑通標誌

每次任務執行完，走這個閉環：

1. **反饋回寫**：把任務結果/修正寫回資料層或規則層，讓下次更準
2. **人工驗收**：人工檢查輸出是否符合預期
3. **跑通標誌**：確認以下三點都成立，才算這條工作流「跑通」
   - [ ] 能找到資料
   - [ ] 能標注來源
   - [ ] 改動量變少

## 建議做法

每個真實任務建一份工作流定義，放在這個資料夾，例如：

- `write-article.md`
- `customer-qa.md`
- `sales-script.md`
- `onboarding-training.md`
- `retro.md`

每份工作流定義應該說明：用到哪些資料層分類、走哪個索引層方式、套用哪份規則檔、驗收標準是什麼。
