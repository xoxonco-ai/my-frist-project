# 資料層：AI 能讀什麼

## 原則：先乾淨，再變多

AI 只讀「乾淨資料」，不讀「髒亂資料」。

### 髒亂資料（不要放進來）

- 圖片 PDF（無法擷取文字）
- 過期版本（如 合同_v1、合同_舊版）
- 來源不清（截圖、語音、無出處的內容）
- 排版混亂的掃描件

### 乾淨資料（放進這裡）

資料要同時滿足：**乾淨、可讀、可追溯**。

建議分類（依專案調整）：

- `company/` 公司資料
- `product/` 產品服務
- `faq/` 客戶問題
- `case-studies/` 案例資料
- `meeting-notes/` 會議紀錄
- `worklog/` 工作日記
- `articles/` 歷史文章

### 格式建議

- 一律使用 Markdown
- 語音/影片先轉寫成文字
- 只放真實業務資料，不放測試/範例資料混充
- 標注安全等級（哪些資料不可外流）

### 數量建議

初期抓 **10–30 份**乾淨資料即可，不必求多。資料量大了以後再考慮索引層的進階做法（見 `../index/`）。
